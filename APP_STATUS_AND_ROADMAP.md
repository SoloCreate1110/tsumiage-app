# アプリ状況・機能ロードマップ

最終更新: 2026-05-10

このファイルは、現在のアプリの状態、実装済み機能、確認済みの課題、今後追加したい機能を一箇所で確認するための管理メモです。

## アプリ概要

このアプリは、日々の学習・運動・読書・片付けなどを「積み上げ」として記録する習慣化アプリです。

記録タイプは大きく2種類あります。

- 時間タイプ: タイマーやポモドーロで作業時間を記録する
- 回数タイプ: 回数、冊数、個数、問数などを記録する

記録した内容は、ホーム、詳細、統計、カレンダー、名言リストで確認できます。データは現在、端末内の AsyncStorage に保存されています。

## 技術構成

- Expo / React Native / Expo Router
- TypeScript
- AsyncStorage によるローカル保存
- expo-notifications による通知
- expo-av による効果音
- react-native-draggable-flatlist による並び替え
- react-native-svg による簡易グラフ
- Vitest によるテスト
- サーバー側には Express / tRPC / Drizzle / MySQL 用の構成あり

## 現在の主要画面

### ホーム

- 今日の日付と挨拶を表示
- 積み上げ項目の一覧を表示
- 今日の記録量を項目ごとに表示
- 連続日数を表示
- 項目の長押し並び替え
- 実行中タイマーがある場合、ライブバナーを表示
- 今日の名言を表示
- 名言カードから名言リストへ移動
- FAB から新規項目を追加
- 設定画面への導線

### 新規項目追加

- 項目名を入力
- 時間タイプ / 回数タイプを選択
- 回数タイプは単位を設定可能
- アイコンを選択
- カラーを選択
- プレビュー表示
- 保存時に項目を追加

### 項目詳細

- 項目ごとの累計値を表示
- 時間タイプは通常タイマーで記録
- 時間タイプはポモドーロ記録に対応
- ポモドーロは 25分作業 / 5分休憩
- タイマー停止時に経過時間を記録
- 回数タイプはカウンターで記録
- 記録時に作業ログ用メモを付けられる
- 今日のメモを自動保存
- 直近の作業ログを表示
- 最近の記録を日付ごとに集計表示
- 日別記録を後から増減調整
- 目標の進捗、残り、今日の目安を表示
- 項目ごとにリマインダー通知を最大5件まで設定

### 目標設定

- 項目名、アイコン、カラーを編集
- 目標値を設定
- 期限を設定
- 現在の累計から、期限までの1日あたり目安を計算
- 項目削除

### 統計

- 全体または項目ごとに表示対象を切り替え
- 合計時間を表示
- 合計回数を表示
- 連続日数を表示
- 記録数を表示
- 直近7日間の時間グラフを表示
- 直近7日間の回数グラフを表示

### カレンダー

- 全体または項目ごとの活動記録をヒートマップで表示
- 選択した日の記録サマリーを表示
- 今年の残り週数を表示
- 活動日数のサマリーを表示
- 表示内容を共有するボタンあり

### 名言リスト

- 今日表示された名言を収集済みとして管理
- 収集済み件数を表示
- 未収集の名言は伏せて表示
- 収集履歴のリセット

### 設定

- 通知のオン / オフ
- 効果音の音量調整
- アプリ内お知らせ
- SNS / 問い合わせリンク
- アプリバージョン表示

積み上げ記録です。

- id
- itemId
- value
- date
- createdAt
- note

### DailyNote

- itemId
- date
- text
- updatedAt

## 確認済みの課題

- 一部の画面テキストやコメントに文字化けが残っている
- 既存の `todo.md` と `design.md` も文字化けしており、仕様確認用ドキュメントとして読みづらい
- `edit-item/[id].tsx` には回数単位の初期値などに文字化けが混ざっている
- `calendar.tsx` と `settings.tsx` にユーザー表示文言の文字化けが多い
- `package.json` のアプリ名が `app-template` のまま
- サーバー / 認証 / DB 構成はあるが、現在の主要な積み上げデータはローカル保存中心
- Git は現在の実行ユーザーから見ると safe.directory 未設定で状態確認できない
- テストファイルにも文字化けした説明文が残っている

## 直近で優先したい改善

1. 文字化けしている表示文言の修正
2. `package.json`、アプリ名、README 相当の情報整理
3. 既存の `todo.md` / `design.md` をこのファイルに統合するか、読める文字コードで再作成
4. カレンダー画面と設定画面の表示文言チェック
5. 編集画面の単位初期値を正常な「回」に修正
6. 既存テストの説明文を読みやすい日本語に修正
7. `pnpm test` と `pnpm check` が通る状態を維持
